## 中介(mediator)模式

### 模式简介
```
Mediator 模式是一种行为型设计模式，主要用于减少多个对象或类之间的通信复杂性。
它通过引入一个中介者对象来封装对象之间的交互，使得对象彼此不再需要显式地相互引用，从而实现松耦合。
这有助于简化对象间的依赖关系，提升代码的可维护性和可扩展性。

-结构
Mediator 模式通常涉及以下角色：
中介者接口 (Mediator)：
定义了与组件对象通信的方法，用于协调组件之间的交互。

具体中介者 (ConcreteMediator)：
实现 Mediator 接口，管理并协调多个组件对象之间的交互。

组件 (Colleague)：
表示与其他组件进行交互的对象。组件之间不直接通信，而是通过中介者进行间接通信。

-使用场景
需要集中控制复杂的对象交互，避免对象之间的直接依赖。
系统中存在多个相互依赖的对象，需要解耦它们。
需要引入一个对象来管理多个对象之间的关系，减少系统的复杂性。
```
